# DevOps & AI Hub

A two-part system that aggregates DevOps, SRE, Cloud, and AI/LLM news articles and job openings into a fast, static web app hosted on GitHub Pages.

## Architecture

```
┌─────────────────────────────────────────────────────────────┐
│ Container Worker (Docker, runs via cron / GitHub Actions)   │
│                                                             │
│  News Collectors   Jobs Collectors   LLM Scoring           │
│  (HN, RSS, Dev.to, (RemoteOK, Lever, (OpenRouter,          │
│   Reddit, GitHub)   Greenhouse, ...)  summarise + tag)     │
│                      │                                      │
│              SQLite (dedupe, history)                       │
│                      │                                      │
│         news.json / jobs.json / meta.json                   │
│                      │                                      │
│              git commit + push                              │
└──────────────────────┬──────────────────────────────────────┘
                       ▼
        ┌─────────────────────────────┐
        │  GitHub Pages (/docs)       │
        │  Tab 1: News Feed           │
        │  Tab 2: Job Openings        │
        └─────────────────────────────┘
```

**Key principle:** No API keys ever reach the frontend. All authenticated calls happen inside the container. The frontend only reads static JSON.

## Quick Start (Local)

### Prerequisites
- Docker & Docker Compose
- An [OpenRouter](https://openrouter.ai) API key (free tier works)

### Steps

```bash
# 1. Clone the repo
git clone https://github.com/YOUR_ORG/YOUR_REPO.git
cd YOUR_REPO

# 2. Copy and fill in the env file
cp .env.example .env
# Edit .env — set OPENROUTER_API_KEY at minimum

# 3. Run one full collection cycle
docker compose up --build worker

# 4. Preview the frontend
docker compose --profile preview up preview
# Open http://localhost:8080
```

The worker runs once and exits. `docs/data/*.json` will be populated with fresh data.

## GitHub Pages Setup

1. Push the repo to GitHub.
2. Go to **Settings → Pages** → set source to **Deploy from a branch** → branch `main`, folder `/docs`.
3. Add the following **Repository Secret** (Settings → Secrets → Actions):
   - `OPENROUTER_API_KEY` — your OpenRouter key
4. The `worker-schedule.yml` workflow fires every 2 hours, collects data, commits updated JSON, and pushes — which triggers `pages-deploy.yml` to re-deploy the site automatically.

### Manual trigger
Go to **Actions → Refresh Data (Worker Schedule) → Run workflow**.

## Configuration

All source lists and settings live in [`worker/config.yaml`](worker/config.yaml) — **no code changes needed** to add/remove a source.

### Key sections

| Section | What it controls |
|---------|-----------------|
| `retention_days` | How many days of data to keep in SQLite / export |
| `keyword_filter` | Pre-LLM gate — item must contain at least one keyword |
| `llm.model` | OpenRouter model (default: `nvidia/nemotron-3-ultra-550b-a55b:free`) |
| `llm.batch_size` | Items per OpenRouter request (controls cost) |
| `news.*` | Enable/disable each news source, configure tags/feeds |
| `jobs.*` | Enable/disable each job source, add Greenhouse/Lever slugs |

### Adding a news RSS feed
```yaml
news:
  rss_feeds:
    feeds:
      - name: "My Blog"
        url:  "https://example.com/feed.xml"
```

### Adding a Greenhouse or Lever board
```yaml
jobs:
  greenhouse:
    boards:
      - your-company-slug
  lever:
    boards:
      - your-company-slug
```

## Scheduling Options

### Option A — GitHub Actions (default, recommended)
Zero infrastructure. The `worker-schedule.yml` workflow runs on a cron schedule, executes `main.py` directly on the Actions runner, commits JSON, and pushes. See [`.github/workflows/worker-schedule.yml`](.github/workflows/worker-schedule.yml).

### Option B — Self-hosted container
Run the container on a VM, homelab, or Kubernetes CronJob with a host-level cron:

```bash
# /etc/cron.d/devops-news  (runs every 2 hours)
0 */2 * * * root cd /path/to/repo && docker compose up --build worker
```

Set `GH_PAT` and `GIT_REPO_URL` in `.env` so the container can push the updated JSON back to GitHub. See `.env.example` for all variables.

## Environment Variables

| Variable | Required | Description |
|----------|----------|-------------|
| `OPENROUTER_API_KEY` | Yes | OpenRouter API key for LLM scoring |
| `OPENROUTER_MODEL` | No | Model override (default: `nvidia/nemotron-3-ultra-550b-a55b:free`) |
| `GH_PAT` | Option B only | GitHub PAT with `repo` scope for auto-push |
| `GIT_REPO_URL` | Option B only | Full HTTPS clone URL of the repo |
| `GIT_BRANCH` | No | Branch to push to (default: `main`) |
| `LOG_LEVEL` | No | `DEBUG` / `INFO` / `WARNING` (default: `INFO`) |
| `DRY_RUN` | No | Set `true` to collect/score but skip git push |

## Frontend Features

- Two tabs: **News Feed** and **Job Openings**
- Client-side filters: tag/category, source, date range (24 h / 7 d / 30 d), text search
- Pagination — 20 items per page
- "Last updated" timestamp with stale-data warning (> 6 h)
- Dark mode toggle
- Responsive — mobile and desktop
- Graceful empty/error states if JSON is missing or malformed
- Pure vanilla JS — no build step, no npm

```bash
pip install pytest

```

## Non-Functional Guarantees

- **No secrets in frontend** — `docs/` never contains API keys
- **Idempotent runs** — re-running the worker does not duplicate entries
- **Source resilience** — one failing source never crashes the whole run; failures are logged and surfaced in `meta.json.source_health`
- **Rate-limit friendly** — configurable delays for Reddit and GitHub feeds
